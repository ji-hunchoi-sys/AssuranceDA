# AssuranceDA — 회계감사 실무 보조 웹 도구 모음

회계감사 실무에서 자주 쓰이는 작업 두 가지를 단일 저장소로 묶은 도구 세트입니다.

| 도구 | 진입점 | 형태 |
|------|--------|------|
| **DART 감사보고서 검색기** | `index_pdf.html` (백엔드 `app.py`) | Flask + Vanilla JS |
| **자금일보 검증기 Pro** | `index2_pdf.html` | 단일 페이지 HTML (백엔드 불필요) |

## 1. DART 감사보고서 검색기

DART(전자공시시스템)에 공시된 감사보고서·사업보고서를 **본문 키워드**와 **감사인(회계법인)별**로 동시에 필터링해 검색하고, 주석 본문을 앱 안에서 바로 열람할 수 있는 웹 애플리케이션입니다.

> DART 사이트는 감사인별 필터링 기능을 제공하지 않습니다. 본 도구는 사업보고서 XML에서 감사인명을 자동 추출하여 회계감사 실무자가 **타사의 주석 기재 사례**를 빠르게 비교 참고할 수 있도록 합니다.

### 주요 기능

- **본문 키워드 검색** — DART `dsab007/search.ax` 기반, AND/OR/NOT·동의어 검색 지원
- **회사명 자동 매핑** — 입력한 회사명을 DART 회사코드(CIK)로 변환해 정확도 향상
- **보고서 유형 필터** — 정기공시(A) / 외부감사(F) 등 `dsp_type` 단위 다중 선택
- **감사인 자동 추출** — 사업보고서 XML에서 "외부감사에 관한 사항" 섹션을 파싱해 감사인명 인식
- **감사인별 클릭 필터링** — 검색 결과 상단의 감사인 태그 클릭으로 즉시 필터
- **주석 본문 인앱 렌더링** — DART 뷰어 링크 + 섹션 단위 HTML 추출 제공
- **세션·캐시 최적화** — DART 웹 세션 재사용, 회사코드/감사인/XML 캐시로 응답 속도 단축

### 설치 및 실행

```bash
# 1) 의존성 설치
pip install -r requirements.txt

# 2) 환경변수 설정 (.env 생성)
#   → .env 파일을 생성해 DART_API_KEY 값 입력
#   DART_API_KEY=발급받은_키

# 3) 서버 실행
python app.py
#   → http://localhost:5000 접속
```

### 환경변수

| 키 | 설명 |
|----|------|
| `DART_API_KEY` | OpenDART API 키. [opendart.fss.or.kr](https://opendart.fss.or.kr) 에서 무료 발급 |

### API 명세

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET  | `/api/health` | 서버 상태 확인 |
| GET  | `/api/search` | 키워드/회사명 기반 보고서 검색 (`keyword`, `corp_name`, `start_date`, `end_date`, `dsp_type`, `page`, `max_results`) |
| POST | `/api/extract-auditors` | 접수번호 배열에 대해 감사인명 일괄 추출 (`rcept_nos`) |
| GET  | `/api/extract-auditor/<rcept_no>` | 단일 보고서의 감사인명 추출 |
| GET  | `/api/document/<rcept_no>` | 보고서 내 섹션(목차) 목록 |
| GET  | `/api/document/<rcept_no>/content?section=<idx>` | 특정 섹션의 HTML 본문 |

상세 요구사항은 [`PRD.md`](./PRD.md) 참고.

## 2. 자금일보 검증기 Pro

자금팀이 작성한 **자금일보**(일일 자금 보고서)와 실제 **은행 입출금 내역**을 교차 비교해 누락·초과·금액 불일치를 자동 검출하는 도구입니다. 백엔드 없는 단일 HTML 파일이라 사내 PC에서 바로 열어 사용할 수 있습니다.

### 주요 기능

- **다양한 입력 형식** — 자금일보·은행 모두 PDF / Excel(.xlsx, .xls) / CSV / 이미지(PNG, JPG) 지원
- **OCR 내장** — Tesseract.js로 이미지 PDF·캡쳐도 텍스트화 (한국어+영어 모델)
- **PDF 줄 단위 추출** — pdf.js의 텍스트 아이템을 Y좌표로 그룹핑해 표 구조 보존
- **항목명 기반 분류** — `외상대 수금`, `정부지원자금`, `일반경비`, `전도금` 등 키워드로 수입/지출/조달 자동 분류 (OCR 오인식 `정부→성부` 같은 흔한 케이스도 허용)
- **잔액 체인 검증** — 은행 거래 시계열의 잔액 변화로 거래 금액·방향을 역추론·보정
- **합산 매칭(N:1)** — 자금일보 1행이 은행 N건의 합인 경우 부분합(subset sum)으로 자동 매칭
  - 예: 자금일보 `일반경비 8,000,000` ↔ 은행 `사무용품 2M + 통신비 1M + 임대료 5M`
- **OCR 디버그 패널** — 자금일보·은행 각각 OCR/PDF 원문과 분류 결과를 화면에 표시
- **검증 결과 시각화** — 합계 비교, 1:1 매칭, N:1 합산, 누락/초과 항목을 카드 형태로 정리

### 사용 방법

```
1. 브라우저로 index2_pdf.html 직접 열기 (서버 불필요)
2. 좌측 카드에 자금일보 파일 업로드 (PDF/Excel/CSV/PNG/JPG)
3. 우측 카드에 은행 입출금 내역 파일 업로드 (여러 은행 누적 가능)
4. "🔍 대사 검증 시작" 클릭
5. 결과 페이지에서 합계 일치 / 1:1 매칭 / 그룹 합산 / 누락·초과 확인
```

| 단계 | 캡쳐 |
|---|---|
| ① 파일 업로드 + 파싱 결과 디버그 패널 | ![자금일보 검증기 캡쳐1](./자금일보%20검증기%20캡쳐1.png) |
| ② 데이터 미리보기 | ![자금일보 검증기 캡쳐2](./자금일보%20검증기%20캡쳐2.png) |

### 검증 알고리즘 흐름

```
[자금일보] ──► 형식 감지 ──► parseDailyFundPlan / parseFundExcel
                              │
                              ▼ (수입/지출/조달 transactions)
[은행 PDF]  ──► pdf.js (Y좌표 줄 그룹핑) ──► parseBankPDF (잔액 체인)
[은행 PNG]  ──► OpenCV.js 전처리 ──► Tesseract.js OCR ──► parseBankPDF
[은행 Excel]──► XLSX.js ──► parseBankRows (열 자동매핑)
                              │
                              ▼ (개별 transactions)
[크로스 검증]
  Step 1  잔액 검증     기초/마감 시재 일치
  Step 2  합계 검증     수입·지출 합계
  Step 3  그룹 분석     성격별 합산이 은행 합계와 일치하는지
  Step 4  1:1 매칭      자금일보 1건 ↔ 은행 1건
  Step 5  N:1 서브셋합  자금일보 1건 ↔ 은행 N건 부분합 매칭
  Step 6  누락/초과 산출
```

### 자가 회귀 테스트 (`test_samples/`)

OCR·PDF 추출·매칭 로직의 회귀를 막기 위한 자동 테스트 도구를 함께 제공합니다.

```bash
cd test_samples
pip install -r requirements.txt
python -m playwright install chromium

# 시뮬레이션 (Excel/PDF 파서 로직만 Python으로 재현 — 빠름)
python simulate_validator.py     # Excel 경로
python simulate_pdf_path.py      # PDF 경로

# 실제 헤드리스 브라우저로 E2E 테스트 (Tesseract.js OCR까지 검증)
python e2e_test.py
```

E2E 테스트는 `index2_pdf.html`을 헤드리스 Chromium으로 띄워 진짜로 파일을 업로드하고 검증 버튼을 클릭, `APP.results`를 추출해 일치율 100%인지 확인합니다.

### 테스트 시나리오 1 (`자금일보_샘플.xlsx` + `은행거래내역_샘플.xlsx`)

| 구분 | 자금일보 항목 | 금액 | 은행 매칭 |
|---|---|---|---|
| 수입 | 외상대 수금(원화) | 50,000,000 | 1:1 |
| 수입 | 매출 수금 | 30,000,000 | 1:1 |
| 지출 | 외상대 결제 | 20,000,000 | N:1 (8M + 12M) |
| 지출 | 일반경비 | 5,000,000 | N:1 (1.5M + 1.5M + 2M) |
| 지출 | 전도금 | 10,000,000 | 1:1 |

### 테스트 시나리오 2 (`자금일보_샘플2.{pdf,png}` + `은행거래내역_샘플2.{pdf,png}`)

조달(정부지원자금) 섹션 + 입금/출금/조달 모두 포함 + 1:1과 N:1 혼재. 합계 75M 입금, 58M 지출. 시작잔액 200M → 마감잔액 217M.

### 발견·해결한 주요 이슈 (개발 노트)

| # | 문제 | 해결 |
|---|---|---|
| 1 | OCR이 자금일보 표의 세로쓰기 "수입/지출/조달" 헤더를 놓치면 분류 실패 | 항목명 키워드 기반 분류기로 우선 처리, 줄 시작이 `조달/수입/지출`이면 강한 신호로 활용 |
| 2 | OCR이 은행 거래일시 `16:29:29`를 콜론 없이 `162929`로 읽어 시간을 거래금액으로 오인 | 날짜 정규식에 옵션 시간 그룹 추가 + 6자리 stuck timestamp 별도 제거 |
| 3 | OCR이 콤마를 점으로 (`29.536.750`), 또는 위치 어긋남(`29700,000`) | 콤마/마침표 모두 천 단위 구분자로 인정, 4–9자리(미구분) 또는 4–12자리(구분자 포함) 정수만 인정, `0` 시작 미구분자(계좌번호)는 거부 |
| 4 | `parseFundExcel` 헤더 검출이 메타 행("회사명/계획일자")을 헤더로 오인식 | 키워드 매칭 셀 수가 가장 많은 행을 헤더로 채택 + 숫자 셀과 긴 텍스트 셀 제외 |
| 5 | `pdf.js` `getTextContent` 가 모든 텍스트를 한 줄로 합쳐 표 구조 손실 | Y좌표 기반 줄 그룹핑 함수 `extractPdfTextLineAware` 도입 |
| 6 | OCR `정부→성부` 오인식으로 funding 분류 실패 | 분류기에 `[정성]부지원자금/지원금/보조금` 패턴 추가 |
| 7 | 자금일보 메타("회사명: ... 단위: 원")·주석("※ ...") 행에서 가짜 거래 생성 | 메타·주석 패턴 줄을 명시적으로 스킵 |
| 8 | 은행 PDF 첫 줄 입출금 방향이 무조건 출금으로 잘못 분류 | 잔액 변화(`balance > nextBalance` → 입금) + 줄 텍스트 키워드(`입금/수금/회수`) 기반으로 보정 |
| 9 | 은행 Excel에 합계 행이 있으면 거래로 잡혀 합계가 두 배가 됨 | `parseBankRows`에 `합계/소계/total/이월` 패턴 필터 추가 |

## 라이선스 / 참고

- 본 도구는 회계감사 실무자의 사내 사용을 전제로 합니다.
- DART 검색기는 OpenAPI 이용약관에 따라 요청 간 0.35초 rate limiting 적용
- 자금일보 검증기는 사용자 데이터를 외부로 전송하지 않습니다(브라우저 내 처리)

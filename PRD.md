## 1. 제품 개요 (Product Overview)

### 1.1 제품 비전 (Product Vision)

한국 DART(전자공시시스템)에 공시된 감사보고서를 **감사인(회계법인)별로 필터링하여 검색**하고, 감사보고서 내 **주석(Notes to Financial Statements) 사례를 비교 열람**할 수 있는 웹 애플리케이션을 개발한다.

회계감사 전문가가 타사의 감사보고서 주석 기재 사례를 빠르게 검색하고 참고하여 자사 감사보고서 작성 품질을 향상시키는 것이 이 제품의 핵심 가치이다.

### 1.2 배경 및 맥락 (Background & Context)

| 구분 | 현황 |
|------|------|
| **문제 상황** | 회계 감사인이 타사 감사보고서의 주석 기재 사례를 참고하려면 DART에서 개별 공시를 하나씩 열어봐야 하며, 감사인(회계법인)별 필터링 기능이 DART에 존재하지 않음 |
| **기존 대안** | DART 전자공시 직접 검색 (비효율적), 유료 데이터베이스 (고비용), 수동 엑셀 정리 (시간 소요) |
| **기회** | OpenDart API를 활용하면 감사보고서 목록 조회 시 `flr_nm`(제출인명) 필드에 감사인 정보가 포함되어 있어 감사인별 필터링이 프로그래밍적으로 가능 |
| **제약** | 단일 HTML 파일 + Flask 백엔드 구조, 외부 프레임워크 최소화 |

### 1.3 목표 (Goals & Objectives)

| 목표 ID | 목표 | 측정 기준 | 달성 기한 |
|---------|------|-----------|-----------|
| G-01 | 감사보고서를 감사인별로 필터링하여 검색할 수 있다 | 5대 Big4 + 주요 회계법인 10개 이상 필터링 가능 | MVP |
| G-02 | 감사보고서 주석 본문을 앱 내에서 바로 열람할 수 있다 | 주석 섹션 직접 렌더링 또는 DART 뷰어 링크 제공 | MVP |
| G-03 | 검색부터 주석 열람까지 3클릭 이내에 도달할 수 있다 | 사용자 테스트 기준 평균 3클릭 이하 | MVP |
| G-04 | 여러 회사의 주석 사례를 나란히 비교할 수 있다 | 최소 2개 보고서 병렬 비교 기능 | v1.1 |

### 2.3 사용자 시나리오 (User Scenarios)

#### 시나리오 1: 감사인별 주석 사례 검색 (Primary Flow)

```
1. 사용자가 검색 화면에 접속한다.
2. 기간을 "2024-01-01 ~ 2024-03-31"로 설정한다.
3. 감사인 드롭다운에서 "삼일회계법인"을 선택한다.
4. [검색] 버튼을 클릭한다.
5. 삼일회계법인이 감사한 감사보고서 목록이 표시된다.
6. 특정 회사의 감사보고서를 클릭한다.
7. 하위 문서 목록(주석, 재무제표 등)이 펼쳐진다.
8. "주석"을 클릭하면 DART 뷰어에서 주석 본문이 표시된다.
```

#### 시나리오 2: 회사명 검색 후 감사인 확인

```
1. 사용자가 회사명 검색창에 "삼성전자"를 입력한다.
2. 삼성전자의 감사보고서 목록이 표시된다.
3. 각 보고서의 감사인(제출인)이 함께 표시된다.
4. 사용자가 감사인을 확인하고 주석을 열람한다.
```

#### 시나리오 3: 다수 회사 주석 비교 (v1.1)

```
1. 사용자가 동일 감사인의 보고서 목록에서 여러 회사를 체크박스로 선택한다.
2. [비교하기] 버튼을 클릭한다.
3. 선택한 회사들의 주석이 탭 형태로 나란히 표시된다.
```

### 3.2 상세 기능 명세 (Detailed Feature Specifications)

---

#### F-01: 기간별 감사보고서 검색

**설명**: 사용자가 지정한 시작일과 종료일 범위 내에 DART에 공시된 감사보고서(외부감사관련, kind='F') 목록을 조회한다.

**입력**:
- `start_date` (string, 필수): 검색 시작일 (형식: YYYY-MM-DD)
- `end_date` (string, 필수): 검색 종료일 (형식: YYYY-MM-DD)

**출력**: 감사보고서 목록 (DataFrame/JSON)
- `corp_code`: 고유번호
- `corp_name`: 회사명
- `stock_code`: 종목코드
- `corp_cls`: 법인구분 (Y=유가, K=코스닥, E=기타)
- `report_nm`: 보고서명
- `rcept_no`: 접수번호
- `flr_nm`: 제출인명 (= 감사인/회계법인명)
- `rcept_dt`: 접수일자

**비즈니스 로직**:
1. OpenDartReader의 `list()` 메서드 호출: `dart.list(start=start_date, end=end_date, kind='F')`
2. `kind='F'`는 외부감사관련 공시를 의미하며, 감사보고서와 연결감사보고서가 모두 포함됨
3. 기본 기간 미지정 시 최근 3개월을 기본값으로 설정
4. `final=True` 유지하여 최종 공시만 반환

**수용 기준 (Acceptance Criteria)**:

```
Given: 사용자가 시작일 "2024-01-01", 종료일 "2024-03-31"을 입력한 경우
When: [검색] 버튼을 클릭하면
Then: 해당 기간 내 DART에 공시된 외부감사관련 보고서 목록이 테이블에 표시된다
  And: 각 행에 회사명, 보고서명, 감사인(flr_nm), 접수일이 포함된다
  And: 결과가 접수일 기준 최신순으로 정렬된다

Given: 시작일이 종료일보다 미래인 경우
When: [검색] 버튼을 클릭하면
Then: "시작일은 종료일보다 이전이어야 합니다" 오류 메시지가 표시된다

Given: 검색 결과가 0건인 경우
When: 검색이 완료되면
Then: "검색 결과가 없습니다" 메시지가 표시된다
```

#### F-02: 감사인별 필터링

**설명**: 검색 결과를 감사인(회계법인) 이름 기준으로 필터링한다. OpenDart API 응답의 `flr_nm` 필드를 활용한다.

**입력**:
- `auditor_name` (string, 선택): 감사인 이름. "전체" 선택 시 필터링 없음

**비즈니스 로직**:
1. 검색 결과의 `flr_nm` 필드에서 고유한 감사인 목록을 추출
2. 감사인 목록을 건수 기준 내림차순 정렬하여 드롭다운에 표시
3. 선택된 감사인과 `flr_nm`이 정확히 일치하는 행만 필터링
4. "전체" 선택 시 모든 결과 표시
5. 주요 감사인(Big4) 상단 고정 옵션 제공:
   - 삼일회계법인 (PwC 삼일)
   - 삼정회계법인 (삼정KPMG)
   - 한영회계법인 (EY한영)
   - 안진회계법인 (Deloitte 안진)

**수용 기준 (Acceptance Criteria)**:

```
Given: 검색 결과가 표시된 상태에서
When: 감사인 드롭다운에서 "삼일회계법인"을 선택하면
Then: flr_nm이 "삼일회계법인"인 보고서만 테이블에 표시된다
  And: 감사인 컬럼의 모든 값이 "삼일회계법인"이다
  And: 필터링된 건수가 표시된다

Given: 감사인 필터가 적용된 상태에서
When: "전체"를 선택하면
Then: 필터가 해제되고 전체 검색 결과가 다시 표시된다

Given: 검색 결과가 로드된 상태에서
When: 감사인 드롭다운을 열면
Then: 해당 검색 결과에 포함된 모든 감사인이 목록에 표시된다
  And: Big4 회계법인이 상단에 구분선과 함께 우선 표시된다
  And: 각 감사인 옆에 건수가 (N건) 형태로 표시된다
```

#### F-03: 감사인 드롭다운 목록

**설명**: 검색 결과에서 감사인(회계법인) 목록을 동적으로 추출하여 드롭다운 선택 UI를 제공한다.

**비즈니스 로직**:
1. 검색 API 응답에서 `flr_nm` 고유값 추출
2. 정렬 방식:
   - 1순위: Big4 고정 (삼일 > 삼정 > 한영 > 안진)

3. 각 감사인 옆에 `(N건)` 형태로 건수 표시
4. 첫 번째 옵션은 "전체 (총 N건)"

#### F-04: 회사명 검색

**설명**: 회사명으로 감사보고서를 검색한다.

**입력**:
- `corp_name` (string, 선택): 회사명 (부분 일치 검색)

**비즈니스 로직**:
1. 방법 A (기간 검색 후 클라이언트 필터): `dart.list()` 결과에서 `corp_name`에 입력값이 포함된 행 필터링
2. 방법 B (회사 직접 검색): `dart.list(corp=종목코드)` 로 특정 회사의 공시 목록 조회
3. 회사명은 부분 일치(contains) 방식으로 검색
4. 기간 필터와 함께 사용 가능 (AND 조건)

**수용 기준 (Acceptance Criteria)**:

```
Given: 사용자가 회사명 입력란에 "삼성"을 입력한 경우
When: [검색] 버튼을 클릭하면
Then: corp_name에 "삼성"이 포함된 모든 감사보고서가 표시된다
  And: 감사인 필터와 함께 적용 가능하다

Given: 회사명과 기간을 함께 지정한 경우
When: [검색] 버튼을 클릭하면
Then: 해당 기간 내 해당 회사명을 포함하는 감사보고서만 표시된다
```

#### F-05: 검색 결과 테이블 표시

**설명**: 검색 결과를 정보가 잘 정리된 테이블 형태로 화면에 표시한다.

**테이블 컬럼**:

| 컬럼 | 데이터 소스 | 정렬 | 너비 |
|------|------------|------|------|

| 회사명 | `corp_name` | 가나다순 | 150px |
| 보고서명 | `report_nm` | - | 200px |
| 감사인 | `flr_nm` | 가나다순 | 130px |
| 접수일 | `rcept_dt` | 최신순 (기본) | 100px |

| 상세 | 버튼 | - | 80px |

**법인구분 변환 규칙**:
- `Y` -> "유가증권"
- `K` -> "코스닥"

**수용 기준 (Acceptance Criteria)**:

```
Given: 검색 결과가 100건 이상인 경우
When: 결과가 표시되면
Then: 페이지당 20건씩 표시되며 페이지네이션 컨트롤이 하단에 표시된다
  And: 현재 페이지와 전체 페이지 수가 표시된다

Given: 테이블 컬럼 헤더를 클릭한 경우
When: 클릭하면
Then: 해당 컬럼 기준으로 오름차순/내림차순 토글 정렬된다
```

#### F-06: 하위 문서 목록 조회

**설명**: 특정 감사보고서를 선택하면 해당 보고서의 하위 문서(주석, 재무제표, 감사의견 등) 목록을 조회하여 표시한다.

**비즈니스 로직**:
1. 선택된 보고서의 `rcept_no`를 사용하여 `dart.sub_docs(rcept_no)` 호출
2. 반환되는 DataFrame 컬럼: `title`, `url`
3. 일반적인 하위 문서 구조:
   - 감사보고서 (표지)
   - 독립된 감사인의 감사보고서 (감사의견)
   - (첨부)재무제표
   - **주석** (핵심 타겟)
   - 외부감사 실시내용

**수용 기준 (Acceptance Criteria)**:

```
Given: 검색 결과 테이블에서 특정 보고서의 [상세] 버튼을 클릭한 경우
When: 하위 문서 목록이 로드되면
Then: 해당 보고서의 하위 문서 제목과 링크가 펼침(accordion) 형태로 표시된다
  And: "주석" 항목이 시각적으로 강조(하이라이트)된다

Given: 하위 문서 목록에서 "주석"을 클릭한 경우
When: 클릭하면
Then: DART 뷰어 URL이 새 탭에서 열리거나 인앱 프리뷰로 표시된다
```

#### F-07: 주석 본문 열람

**설명**: 감사보고서 주석 문서를 DART 뷰어를 통해 열람한다.

**비즈니스 로직**:
1. `sub_docs()`에서 반환된 URL 중 제목이 "주석"에 해당하는 URL을 추출
2. URL 형식: `http://dart.fss.or.kr/report/viewer.do?rcpNo=...&dcmNo=...&eleId=...`
3. 열람 방식:
   - 기본: 새 탭에서 DART 뷰어 URL 열기
   - 확장 (F-12): iframe으로 앱 내 표시

**수용 기준 (Acceptance Criteria)**:

```
Given: 하위 문서 목록이 표시된 상태에서
When: "주석" 링크를 클릭하면
Then: DART 뷰어가 새 브라우저 탭에서 열려 주석 본문이 표시된다
  And: 원본 검색 화면은 유지된다

Given: 하위 문서 목록에 "주석" 항목이 없는 경우
When: 하위 문서 목록이 표시되면
Then: "주석 문서를 찾을 수 없습니다" 메시지가 표시된다
```

#### F-09: 보고서 유형 필터

**설명**: 감사보고서와 연결감사보고서를 구분하여 필터링한다.

**비즈니스 로직**:
1. `report_nm` 필드 분석:
   - "감사보고서" 포함 & "연결" 미포함 -> 개별 감사보고서
   - "연결감사보고서" 포함 -> 연결감사보고서
2. 필터 옵션: 전체 / 감사보고서(개별) / 연결감사보고서
3. 감사인 필터와 AND 조건으로 결합

#### F-12: 주석 본문 인앱 프리뷰

**설명**: DART 뷰어 URL을 iframe으로 로드하여 페이지를 이동하지 않고 앱 내에서 주석을 확인한다.

**비즈니스 로직**:
1. 하위 문서 URL을 iframe의 `src`로 설정
2. DART 뷰어가 X-Frame-Options 제한이 있을 수 있으므로, 불가능한 경우 새 탭 열기로 폴백
3. Flask 백엔드에서 프록시 엔드포인트를 제공하여 CORS/X-Frame-Options 우회
   - `/api/proxy?url=<dart_viewer_url>` -> HTML 본문을 프록시하여 반환

**수용 기준 (Acceptance Criteria)**:

```
Given: 하위 문서에서 "주석"의 [미리보기] 버튼을 클릭한 경우
When: 프리뷰가 로드되면
Then: 검색 화면 하단 또는 측면 패널에 주석 본문이 iframe으로 표시된다
  And: 스크롤을 통해 전체 주석 본문을 확인할 수 있다

Given: iframe 로딩이 CORS/보안 정책으로 차단된 경우
When: 로딩 실패가 감지되면
Then: 자동으로 새 탭에서 DART 뷰어를 열고 사용자에게 안내 메시지를 표시한다
```

## 5. 기술 아키텍처 (Technical Architecture)

### 5.1 시스템 구성도

```
┌─────────────────────┐      ┌──────────────────────┐      ┌──────────────┐
│   index_pdf.html    │      │   Flask Backend      │      │  OpenDart    │
│   (Frontend)        │◄────►│   (app.py)           │◄────►│  API Server  │
│                     │ AJAX │                      │HTTP  │              │
│ - Tailwind CSS      │      │ - OpenDartReader     │      │ dart.fss.or  │
│ - Vanilla JS        │      │ - API Proxy          │      │ .or.kr       │
│ - DOM 조작          │      │ - CORS 처리          │      │              │
└─────────────────────┘      │ - 응답 캐싱          │      └──────────────┘
                             └──────────────────────┘
                                       │
                                       │ HTTP Proxy
                                       ▼
                             ┌──────────────────────┐
                             │ DART Report Viewer   │
                             │ (주석 HTML 프록시)    │
                             └──────────────────────┘
```

### 5.2 데이터 흐름

```
1. [사용자] 검색 조건 입력 → [Frontend JS]
2. [Frontend JS] AJAX POST /api/search → [Flask]
3. [Flask] dart.list(start, end, kind='F') → [OpenDart API]
4. [OpenDart API] 응답 (DataFrame) → [Flask]
5. [Flask] JSON 변환, 감사인 목록 추출 → [Frontend JS]
6. [Frontend JS] 테이블 렌더링, 감사인 드롭다운 갱신 → [사용자]

7. [사용자] 특정 보고서 [상세] 클릭 → [Frontend JS]
8. [Frontend JS] AJAX GET /api/subdocs/<rcept_no> → [Flask]
9. [Flask] dart.sub_docs(rcept_no) → [OpenDart API/DART]
10. [DART] 하위문서 목록 → [Flask] → [Frontend JS]
11. [Frontend JS] 하위문서 아코디언 렌더링 → [사용자]

12. [사용자] "주석" 클릭 → [새 탭] DART 뷰어 URL
    또는 → [Frontend JS] /api/proxy?url=<dart_url> → [Flask] → [DART Viewer]
```

### 5.3 기술 스택 상세

| 계층 | 기술 | 버전 | 역할 |
|------|------|------|------|
| **Frontend** | HTML5 | - | 단일 페이지 구조 |
| **Frontend** | Tailwind CSS | CDN (3.x) | 스타일링 |

**Request Body** (JSON):

```json
  "corp_name": "",
  "report_type": "all"
}
```

|---------|------|------|------|
| `start_date` | string | Y | 검색 시작일 (YYYY-MM-DD) |
| `end_date` | string | Y | 검색 종료일 (YYYY-MM-DD) |
| `corp_name` | string | N | 회사명 필터 (부분 일치) |
| `report_type` | string | N | "all" / "audit" / "consolidated" |

**Response** (JSON):
```json
{
  "success": true,
  "total_count": 5438,
  "data": [
    {

      "rcept_no": "20240329003608",
      "flr_nm": "삼일회계법인",
      "rcept_dt": "20240329"

  "auditors": [
    {"name": "삼일회계법인", "count": 469, "is_big4": true},
    {"name": "삼정회계법인", "count": 338, "is_big4": true},
    {"name": "한영회계법인", "count": 366, "is_big4": true},
```

#### `GET /api/proxy`

DART 뷰어 HTML을 프록시하여 반환한다 (CORS/X-Frame-Options 우회용).

**Query Parameter**:
| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `url` | string | DART 뷰어 URL (URL 인코딩 필요) |

**Response**: HTML 본문 (Content-Type: text/html; charset=utf-8)

**보안 고려사항**:
- `url` 파라미터의 도메인이 `dart.fss.or.kr`인지 반드시 검증 (SSRF 방지)
- 허용 도메인 화이트리스트: `['dart.fss.or.kr']`

### 6.2 OpenDart API 사용 매핑

| Flask 엔드포인트 | OpenDartReader 메서드 | 비고 |
|-----------------|---------------------|------|
| `POST /api/search` | `dart.list(start, end, kind='F')` | kind='F'는 외부감사관련 |
| `GET /api/subdocs/<rcept_no>` | `dart.sub_docs(rcept_no)` | 하위 문서 목록 반환 |
| `GET /api/proxy` | `requests.get(url)` | DART 뷰어 HTML 프록시 |

### 6.3 OpenDart API 제약사항

| 항목 | 제약 | 대응 방안 |
|------|------|-----------|
| 일일 호출 한도 | 10,000회/일 | 검색 결과 서버 측 메모리 캐싱 (TTL 30분) |
| 분당 호출 한도 | 약 1,000회/분 | 클라이언트 단 디바운싱 (300ms) |
| 기간별 최대 반환 건수 | 공시 수에 의존 (제한 없음) | 과도한 기간 설정 시 경고 (6개월 초과) |
| `flr_nm` 필드 | 제출인명 = 감사인 (kind='F' 한정) | kind='F'로 한정하여 정확도 보장 |

| 주석 | Notes to Financial Statements | 재무제표의 이해를 돕기 위한 추가 설명 및 상세 정보 |
| 감사인 | Auditor / Audit Firm | 외부감사를 수행하는 공인회계사 또는 회계법인 |
| 접수번호 (rcept_no) | Receipt Number | DART 공시 고유 식별 번호 (14자리) |
| 제출인 (flr_nm) | Filer Name | 공시 제출자명 (외부감사관련 공시에서는 감사인명) |
| K-IFRS | Korean International Financial Reporting Standards | 한국채택 국제회계기준 |
| Big4 | Big Four | 글로벌 4대 회계법인 (PwC, KPMG, EY, Deloitte) |

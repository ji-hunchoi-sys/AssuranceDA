# 자금일보 검증기 Pro

자금팀이 작성한 **자금일보**(일일 자금 보고서)와 실제 **은행 입출금 내역**을 교차 비교해 누락·초과·금액 불일치를 자동 검출하는 도구입니다. 또한 **은행 거래내역만 입력하면 자금일보를 자동 작성**해주는 생성기 기능도 함께 제공합니다. 백엔드 없는 단일 HTML 파일이라 사내 PC에서 브라우저로 바로 열어 사용할 수 있습니다.

> 진입점: **`index2_pdf.html`** — 좌측 메뉴 "📁 파일 업로드"(검증) / "✍️ 자금일보 생성"(자동 작성)

---

## 주요 기능

### 1) 자금일보 ↔ 은행 대사 검증

- **다양한 입력 형식** — 자금일보·은행 모두 PDF / Excel(.xlsx, .xls) / CSV / 이미지(PNG, JPG) 지원
- **다중 파일 동시 업로드** — 여러 은행 파일을 한 번에 또는 순차로 업로드 (다중 선택/누적)
- **OCR 내장** — Tesseract.js로 이미지·캡쳐 PDF도 텍스트화 (한국어+영어 모델, 자동 한국어 학습 데이터 다운로드)
- **PDF 줄 단위 추출** — pdf.js 텍스트 아이템을 Y좌표로 그룹핑해 표 구조 보존
- **항목명 기반 분류** — `외상대 수금`, `정부지원자금`, `일반경비`, `전도금` 등 키워드로 수입/지출/조달 자동 분류 (OCR 오인식 `정부→성부` 같은 흔한 케이스도 허용)
- **잔액 체인 검증** — 은행 거래 시계열의 잔액 변화로 거래 금액·방향을 역추론·보정
- **합산 매칭(N:1)** — 자금일보 1행이 은행 N건의 합인 경우 부분합(subset sum)으로 자동 매칭
  - 예: 자금일보 `일반경비 8,000,000` ↔ 은행 `사무용품 2M + 통신비 1M + 임대료 5M`
- **다중 계좌 시재 합산** — 자금일보가 다중 은행 합산 시재(예: 4개 계좌 합)로 작성된 경우, 각 은행별 첫·마지막 잔액을 합산해 자동 비교
- **OCR 디버그 패널** — 자금일보·은행 각각 OCR/PDF 원문과 분류 결과를 화면에 표시
- **검증 결과 시각화** — 합계 비교, 1:1 매칭, N:1 합산, 누락/초과 항목을 카드 형태로 정리

### 2) 자금일보 자동 생성 (은행 거래내역 → 자금일보)

은행 입출금 내역 파일만 있으면 자금일보를 자동으로 작성해 줍니다. 여러 은행을 한 번에 처리해 통합 자금일보 1건으로 출력합니다.

- **다중 은행 한 번에 업로드** — 4개 은행 거래내역을 한 번 클릭에 다중 선택 (또는 누적 추가)
- **포맷 3종 선택** —
  - **요약형**: 계좌별 합계(전일·당일잔액) + 주요 거래(상위 20%) + 거래 상세
  - **시계열 Ledger형**: 시간순 한 줄 한 거래 (자금일계표)
  - **분류형**: 거래 적요 자동 카테고리(매출수금/외상대 결제/인건비/세금·공과/임대료/통신비 등) + 수입·지출 그룹별 표 + 금일 순증감
- **주요 거래 자동 강조** — 상위 20%/10% 또는 직접 입력 임계금액으로 ⭐ 표시
- **계좌별 자동 그룹화** — 은행별 전일잔액 → 입금 → 출금 → 당일잔액 자동 계산
- **Excel / CSV / 인쇄 출력** — 생성된 자금일보를 그대로 다운로드 또는 인쇄

## 사용 방법

### A. 자금일보 ↔ 은행 대사 검증

```
1. 브라우저로 index2_pdf.html 직접 열기 (서버 불필요)
2. 좌측 카드에 자금일보 파일 업로드 (PDF/Excel/CSV/PNG/JPG)
3. 우측 카드에 은행 입출금 내역 파일 업로드 (여러 은행을 한 번에 또는 누적 가능)
4. "🔍 대사 검증 시작" 클릭
5. 결과 페이지에서 합계 일치 / 1:1 매칭 / 그룹 합산 / 누락·초과 확인
```

### B. 자금일보 자동 생성

```
1. 좌측 메뉴에서 "✍️ 자금일보 생성" 클릭
2. 은행 거래내역 파일 업로드 (PDF/Excel/CSV/이미지) — 한 번에 여러 은행 선택 가능
3. 보고 기준일 / 회사명 / 전일 잔액 / 금액 단위 입력
4. 주요 거래 기준(상위 20% 등) + 📋 자금일보 포맷(요약/시계열 Ledger/분류) 선택
5. "✍️ 자금일보 생성" 클릭 → 결과 화면에서 Excel/CSV/인쇄 다운로드
```

## 검증 알고리즘 흐름

```
[자금일보] ──► 형식 감지 ──► parseDailyFundPlan / parseFundExcel
                              │
                              ▼ (수입/지출/조달 transactions)
[은행 PDF]  ──► pdf.js (Y좌표 줄 그룹핑) ──► parseBankPDF (잔액 체인)
[은행 PNG]  ──► OpenCV.js 전처리 ──► Tesseract.js OCR ──► parseBankPDF
[은행 Excel]──► XLSX.js ──► parseBankRows (열 자동매핑)
                              │
                              ▼ (개별 transactions)
[크로스 검증]
  Step 1  잔액 검증     기초/마감 시재 일치
  Step 2  합계 검증     수입·지출 합계
  Step 3  그룹 분석     성격별 합산이 은행 합계와 일치하는지
  Step 4  1:1 매칭      자금일보 1건 ↔ 은행 1건
  Step 5  N:1 서브셋합  자금일보 1건 ↔ 은행 N건 부분합 매칭
  Step 6  누락/초과 산출
```

## 자가 회귀 테스트 (`test_samples/`)

OCR·PDF 추출·매칭 로직의 회귀를 막기 위한 자동 테스트 도구를 함께 제공합니다.

```bash
cd test_samples
pip install -r requirements.txt
python -m playwright install chromium

# 시뮬레이션 (Excel/PDF 파서 로직만 Python으로 재현 — 빠름)
python simulate_validator.py     # Excel 경로
python simulate_pdf_path.py      # PDF 경로

# 실제 헤드리스 브라우저로 E2E 테스트 (Tesseract.js OCR까지 검증)
python e2e_test.py
```

E2E 테스트는 `index2_pdf.html`을 헤드리스 Chromium으로 띄워 진짜로 파일을 업로드하고 검증 버튼을 클릭, `APP.results`를 추출해 일치율 100%인지 확인합니다.

### 테스트 시나리오 1 (`자금일보_샘플.xlsx` + `은행거래내역_샘플.xlsx`)

| 구분 | 자금일보 항목 | 금액 | 은행 매칭 |
|---|---|---|---|
| 수입 | 외상대 수금(원화) | 50,000,000 | 1:1 |
| 수입 | 매출 수금 | 30,000,000 | 1:1 |
| 지출 | 외상대 결제 | 20,000,000 | N:1 (8M + 12M) |
| 지출 | 일반경비 | 5,000,000 | N:1 (1.5M + 1.5M + 2M) |

## 발견·해결한 주요 이슈 (개발 노트)

| # | 문제 | 해결 |
|---|---|---|
| 1 | OCR이 자금일보 표의 세로쓰기 "수입/지출/조달" 헤더를 놓치면 분류 실패 | 항목명 키워드 기반 분류기로 우선 처리, 줄 시작이 `조달/수입/지출`이면 강한 신호로 활용 |
| 2 | OCR이 은행 거래일시 `16:29:29`를 콜론 없이 `162929`로 읽어 시간을 거래금액으로 오인 | 날짜 정규식에 옵션 시간 그룹 추가 + 6자리 stuck timestamp 별도 제거 |
| 3 | OCR이 콤마를 점으로 (`29.536.750`), 또는 위치 어긋남(`29700,000`) | 콤마/마침표 모두 천 단위 구분자로 인정, 4–9자리(미구분) 또는 4–12자리(구분자 포함) 정수만 인정, `0` 시작 미구분자(계좌번호)는 거부 |
| 4 | `parseFundExcel` 헤더 검출이 메타 행("회사명/계획일자")을 헤더로 오인식 | 키워드 매칭 셀 수가 가장 많은 행을 헤더로 채택 + 숫자 셀과 긴 텍스트 셀 제외 |
| 5 | `pdf.js` `getTextContent` 가 모든 텍스트를 한 줄로 합쳐 표 구조 손실 | Y좌표 기반 줄 그룹핑 함수 `extractPdfTextLineAware` 도입 |
| 6 | OCR `정부→성부` 오인식으로 funding 분류 실패 | 분류기에 `[정성]부지원자금/지원금/보조금` 패턴 추가 |
| 7 | 자금일보 메타("회사명: ... 단위: 원")·주석("※ ...") 행에서 가짜 거래 생성 | 메타·주석 패턴 줄을 명시적으로 스킵 |
| 8 | 은행 PDF 첫 줄 입출금 방향이 무조건 출금으로 잘못 분류 | 잔액 변화(`balance > nextBalance` → 입금) + 줄 텍스트 키워드(`입금/수금/회수`) 기반으로 보정 |
| 9 | 은행 Excel에 합계 행이 있으면 거래로 잡혀 합계가 두 배가 됨 | `parseBankRows`에 `합계/소계/total/이월` 패턴 필터 추가 |

## 비고

- 본 도구는 사용자 데이터를 외부로 전송하지 않습니다. 모든 파싱·OCR·검증은 브라우저 내에서 처리됩니다.
- Tesseract.js 한국어 학습 데이터는 첫 OCR 실행 시 1회 자동 다운로드됩니다(약 10MB).
- 상세 LLM/규칙 기반 분리 권고는 [`AI_RECOMMENDATION.md`](./AI_RECOMMENDATION.md) 참고.
